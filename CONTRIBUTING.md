# Contributing

Thanks for helping improve Agent Studio.

## Local setup

Clone the repo, install dependencies with `pnpm install`, and run the demo or app shell using the workspace scripts in the root `package.json`.

## What to work on

- demo workflows and seeded data
- shared contracts and schemas
- adapters that map runtime data into the shared contract
- docs, examples, and small fixes that improve the public launch

## Before opening a PR

- keep the change focused
- add or update tests when behavior changes
- verify the relevant demo, contract, or adapter path locally
- write a clear PR description that explains what changed and why

## Good PRs

Small, readable, and easy to review. If your change affects public behavior or schema shape, call that out directly.
